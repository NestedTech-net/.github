# Contributing

Thanks for your interest in contributing.

## Before You Start
- Review `docs/public-overview.md`
- Keep contributions aligned with public scope and documentation safety boundaries

## What We Accept Here
- Public documentation improvements
- Clarity/grammar fixes
- Link and policy updates

## What We Do Not Accept Here
- Private architecture details
- Operational secrets or deployment internals
- Sensitive research implementation details

## Contribution Process
1. Open an issue describing the change.
2. Submit a pull request with focused scope.
3. Keep changes minimal and clearly documented.

## Commit and PR Guidance
- Use clear commit messages
- Include rationale in PR description
- Avoid bundling unrelated changes
